# Fogchaser MVP Retrospective
*Date: 2026-03-07*

## What went well

**The model is real.** 6.6x better than random, trained on 6 years of data, with proper calibration and threshold tuning. That's not a toy — it's a functioning ML pipeline.

**Shipping discipline.** Every time we hit a wall (GitHub blocked, Vercel not connecting, bilinear resampling producing garbage values), we found a workaround and kept moving. Manual `fogrefresh`, manual deploys — friction, but not blockers.

**The architecture is clean.** R2 for data, Vercel for the app, completely decoupled. That decision means data refreshes don't require deploys. It was confusing at first but it was the right call.

**The spatial layer is genuinely interesting.** IDW in log-odds space with a terrain offset grid is not a basic feature. It works.

---

## What didn't go well

**Model training sessions were too dense.** Too much explained forward without checking if it was landing. Hard to keep up — that's a process failure, not a comprehension failure.

**GitHub Actions being blocked is unresolved.** Manual `fogrefresh` works but it's friction every time you want fresh data. We never found a proper fix.

**The fog badge bug surfaced late.** The score was wrong for a while before we caught it (peak hour vs. 12-hr average, bilinear resampling garbage). More end-to-end testing earlier would have caught this.

---

## What we learned

- Bilinear resampling at raster borders produces intermediate garbage values — filter by valid range `[0,1]`, not just `> -9990`
- Always output GeoTIFFs in EPSG:4326 — CRS mismatch causes silent null returns from geoblaze
- The month formula matters: `month / 12` not `(month-1) / 12` — a small bug with real model impact
- IEM returns station IDs without K prefix, and renamed the elevation column — external APIs change, always validate
- Badge scoring must use peak hour, not average — averaging buries real fog windows
