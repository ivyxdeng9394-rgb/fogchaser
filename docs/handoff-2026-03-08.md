# Fogchaser Handoff — 2026-03-08

## What We Built This Session

### 1. Fixed fog badge score (critical bug)
**Problem:** Badge always showed "No fog expected" even when map had High/Extreme tiles.
Two root causes:
- `avg_prob` in manifest was ~-43 because bilinear reprojection created garbage border pixels (-8726 etc.) that passed the old `> -9990` filter in `read_geotiff_mean`
- Badge used 12-hour *average* of avg_prob, burying peak fog windows

**Fixes:**
- `read_geotiff_mean` now filters `[0, 1]` range only
- `spatial_pipeline.py` clamps out-of-range pixels to nodata after reprojection
- Badge now uses peak hour's `avg_prob` (not 12-hr average)
- `pixelValuesToColorFn` filter tightened to `v < 0`

### 2. Terrain bullet in "Why this score"
Terrain offset GeoTIFF reprojected to EPSG:4326, uploaded to R2 as `terrain_offset.tif` (static, never changes). On map click, `geoblaze.identify()` reads the terrain offset at that location and adds a location-specific bullet explaining valley/flat/ridge effect. Preloaded at startup so first click is instant.

Terrain offset thresholds (log-odds scale):
- `> 0.25` → "Low-lying valley or sheltered hollow — cold air pools here"
- `> 0.10` → "Low-lying terrain — slightly more fog-prone"
- `>= -0.05` → "Average terrain — no strong terrain effect"
- `< -0.05` → "Elevated or open area — fog less likely"

### 5. Tile seam fix (in progress — needs local verification)
**Problem:** GeoRasterLayer renders in tiles, each canvas has sub-pixel seams → visible light-blue grid over map.
**Solution:** Replaced GeoRasterLayer entirely with a single `L.imageOverlay`. GeoTIFF is rendered pixel-by-pixel to a single canvas, then displayed as one `<img>` covering the full bounds. Zero tile seams possible.
- `georasterToOverlay(gr)` function in app.js handles this
- `geoblaze.identify(gr, ...)` still works — uses the raw georaster object, not the overlay
- Cross-fade updated to use `._image` instead of `._container`

**Status:** Code is written but NOT yet committed or deployed — pending local verification that seams are gone.

## Lessons Learned This Session
- GeoTIFF avg_prob filter must be `[0, 1]` not `> -9990` — bilinear reprojection creates garbage border pixels in valid range
- Badge should use peak hour, not 12-hr average — averaging buries real fog windows
- GeoRasterLayer tile seams can't be fixed with CSS/padding — replace with single `L.imageOverlay`
- `overflow: hidden` on body does not prevent scroll on iOS Safari — use `position: fixed` trick
- `display: block` on canvas is necessary but not sufficient for seam removal
- Always test with mock data locally before deploying — real forecast may have no fog to inspect
